31 /// variable, return lower/upper bounds via the second/third parameter and the

38 /// Try to canonicalize an min/max operations in the context of for `loops` with
39 /// a known range.
40 ///
41 /// `map` is the body of the min/max operation and `operands` are the SSA values
42 /// that the dimensions and symbols are bound to; dimensions are listed first.
43 /// If `isMin`, the operation is a min operation; otherwise, a max operation.
44 /// `loopMatcher` is used to retrieve loop bounds and the step size for a given
45 /// iteration variable.
46 ///
47 /// Note: `loopMatcher` allows this function to be used with any "for loop"-like
48 /// operation (scf.for, scf.parallel and even ops defined in other dialects).
49 LogicalResult canonicalizeMinMaxOpInLoop(RewriterBase &rewriter, Operation *op,
50  AffineMap map, ValueRange operands,
51  bool isMin, LoopMatcherFn loopMatcher);

53 /// Try to simplify a min/max operation `op` after loop peeling. This function
54 /// can simplify min/max operations such as (ub is the previous upper bound of
55 /// the unpeeled loop):
56 /// ```
57 /// #map = affine_map<(d0)[s0, s1] -> (s0, -d0 + s1)>
58 /// %r = affine.min #affine.min #map(%iv)[%step, %ub]
59 /// ```
60 /// and rewrites them into (in the case the peeled loop):
61 /// ```
62 /// %r = %step
63 /// ```
64 /// min/max operations inside the partial iteration are rewritten in a similar
65 /// way.
66 LogicalResult rewritePeeledMinMaxOp(RewriterBase &rewriter, Operation *op,
67  AffineMap map, ValueRange operands,
68  bool isMin, Value iv, Value ub, Value step,
69  bool insideLoop);
