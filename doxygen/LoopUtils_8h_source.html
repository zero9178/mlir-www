171 /// Performs explicit copying for the contiguous sequence of operations in the
172 /// block iterator range [`begin', `end'), where `end' can't be past the

175 /// the output argument `copyNests` is the set of all copy nests inserted, each

231 /// Maps `forOp` for execution on a parallel grid of virtual `processorIds` of
232 /// size given by `numProcessors`. This is achieved by embedding the SSA values
233 /// corresponding to `processorIds` and `numProcessors` into the bounds and step
234 /// of the `forOp`. No check is performed on the legality of the rewrite, it is
235 /// the caller's responsibility to ensure legality.
236 ///
237 /// Requires that `processorIds` and `numProcessors` have the same size and that
238 /// for each idx, `processorIds`[idx] takes, at runtime, all values between 0
239 /// and `numProcessors`[idx] - 1. This corresponds to traditional use cases for:
240 /// 1. GPU (threadIdx, get_local_id(), ...)
241 /// 2. MPI (MPI_Comm_rank)
242 /// 3. OpenMP (omp_get_thread_num)
243 ///
244 /// Example:
245 /// Assuming a 2-d grid with processorIds = [blockIdx.x, threadIdx.x] and
246 /// numProcessors = [gridDim.x, blockDim.x], the loop:
247 ///
248 /// ```
249 /// scf.for %i = %lb to %ub step %step {
250 /// ...
251 /// }
252 /// ```
253 ///
254 /// is rewritten into a version resembling the following pseudo-IR:
255 ///
256 /// ```
257 /// scf.for %i = %lb + %step * (threadIdx.x + blockIdx.x * blockDim.x)
258 /// to %ub step %gridDim.x * blockDim.x * %step {
259 /// ...
260 /// }
261 /// ```
262 void mapLoopToProcessorIds(scf::ForOp forOp, ArrayRef<Value> processorId,
263  ArrayRef<Value> numProcessors);
