23 /// Kind of identifier. Implementation wise SetDims are treated as Range
24 /// ids, and spaces with no distinction between dimension ids are treated
25 /// as relations with zero domain ids.
26 enum class IdKind { Symbol, Local, Domain, Range, SetDim = Range };

28 /// PresburgerSpace is the space of all possible values of a tuple of integer
29 /// valued variables/identifiers. Each identifier has one of the three types:
30 ///
31 /// Dimension: Ordinary variables over which the space is represented.
32 ///
33 /// Symbol: Symbol identifiers correspond to fixed but unknown values.
34 /// Mathematically, a space with symbolic identifiers is like a
35 /// family of spaces indexed by the symbolic identifiers.
36 ///
37 /// Local: Local identifiers correspond to existentially quantified variables.
38 /// For example, consider the space: `(x, exists q)` where x is a dimension
39 /// identifier and q is a local identifier. Let us put the constraints:
40 /// `1 <= x <= 7, x = 2q`
41 /// on this space to get the set:
42 /// `(x) : (exists q : q <= x <= 7, x = 2q)`.
43 /// An assignment to symbolic and dimension variables is valid if there
44 /// exists some assignment to the local variable `q` satisfying these
45 /// constraints. For this example, the set is equivalent to {2, 4, 6}.
46 /// Mathematically, existential quantification can be thought of as the result
47 /// of projection. In this example, `q` is existentially quantified. This can be
48 /// thought of as the result of projecting out `q` from the previous example,
49 /// i.e. we obtained {2, 4, 6} by projecting out the second dimension from
50 /// {(2, 1), (4, 2), (6, 2)}.
51 ///
52 /// Dimension identifiers are further divided into Domain and Range identifiers
53 /// to support building relations.
54 ///
55 /// Identifiers are stored in the following order:
56 /// [Domain, Range, Symbols, Locals]
57 ///
58 /// A space with no distinction between types of dimension identifiers can
59 /// be implemented as a space with zero domain. IdKind::SetDim should be used
60 /// to refer to dimensions in such spaces.
61 ///
62 /// Compatibility of two spaces implies that number of identifiers of each kind
63 /// other than Locals are equal. Equality of two spaces implies that number of
64 /// identifiers of each kind are equal.
