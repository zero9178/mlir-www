45 /// Rewrite a for loop with bounds/step that potentially do not divide evenly
46 /// into a for loop where the step divides the iteration space evenly, followed
47 /// by another scf.for for the last (partial) iteration (if any; returned via
48 /// `partialIteration`). This transformation is called "loop peeling".
49 ///
50 /// This transformation is beneficial for a wide range of transformations such
51 /// as vectorization or loop tiling: It enables additional canonicalizations
52 /// inside the peeled loop body such as rewriting masked loads into unmaked
53 /// loads.
54 ///
55 /// E.g., assuming a lower bound of 0 (for illustration purposes):
56 /// ```
57 /// scf.for %iv = %c0 to %ub step %c4 {
58 /// (loop body)
59 /// }
60 /// ```
61 /// is rewritten into the following pseudo IR:
62 /// ```
63 /// %newUb = %ub - (%ub mod %c4)
64 /// scf.for %iv = %c0 to %newUb step %c4 {
65 /// (loop body)
66 /// }
67 /// scf.for %iv2 = %newUb to %ub {
68 /// (loop body)
69 /// }
70 /// ```
71 ///
72 /// After loop peeling, this function tries to simplify/canonicalize affine.min
73 /// and affine.max ops in the body of the peeled loop and in the body of the
74 /// partial iteration loop, taking advantage of the fact that the peeled loop
75 /// has only "full" iterations. This canonicalization is expected to enable
76 /// further canonicalization opportunities through other patterns.
77 ///
78 /// The return value indicates whether the loop was rewritten or not. Loops are
79 /// not rewritten if:
80 /// * Loop step size is 1 or
81 /// * Loop bounds and step size are static, and step already divides the
82 /// iteration space evenly.
83 ///
84 /// Note: This function rewrites the given scf.for loop in-place and creates a
85 /// new scf.for operation for the last iteration. It replaces all uses of the
86 /// unpeeled loop with the results of the newly generated scf.for.
87 LogicalResult peelAndCanonicalizeForLoop(RewriterBase &rewriter, ForOp forOp,
88  scf::ForOp &partialIteration);

146 /// Populate patterns for SCF software pipelining transformation.
147 /// This transformation generates the pipelined loop and doesn't do any
148 /// assumptions on the schedule dictated by the option structure.
149 /// Software pipelining is usually done in two part. The first part of
150 /// pipelining is to schedule the loop and assign a stage and cycle to each
151 /// operations. This is highly dependent on the target and is implemented as an
152 /// heuristic based on operation latencies, and other hardware characteristics.
153 /// The second part is to take the schedule and generate the pipelined loop as
154 /// well as the prologue and epilogue. It is independent of the target.
155 /// This pattern only implement the second part.
156 /// For example if we break a loop into 3 stages named S0, S1, S2 we would
157 /// generate the following code with the number in parenthesis the iteration
158 /// index:
159 /// S0(0) // Prologue
160 /// S0(1) S1(0) // Prologue
161 /// scf.for %I = %C0 to %N - 2 {
162 /// S0(I+2) S1(I+1) S2(I) // Pipelined kernel
163 /// }
164 /// S1(N) S2(N-1) // Epilogue
165 /// S2(N) // Epilogue
166 void populateSCFLoopPipeliningPatterns(RewritePatternSet &patterns,
167  const PipeliningOption &options);
