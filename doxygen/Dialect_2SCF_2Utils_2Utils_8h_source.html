40 /// Create a clone of `loop` with `newIterOperands` added as new initialization
41 /// values and `newYieldedValues` added as new yielded values. The returned
42 /// ForOp has `newYieldedValues.size()` new result values. The `loop` induction
43 /// variable and `newIterOperands` are remapped to the new induction variable
44 /// and the new entry block arguments respectively.
45 ///
46 /// Additionally, if `replaceLoopResults` is true, all uses of
47 /// `loop.getResults()` are replaced with the first `loop.getNumResults()`
48 /// return values respectively. This additional replacement is provided as a
49 /// convenience to update the consumers of `loop`, in the case e.g. when `loop`
50 /// is soon to be deleted.
51 ///
52 /// Return the cloned loop.
53 ///
54 /// This convenience function is useful to factorize common mechanisms related
55 /// to hoisting roundtrips to memory into yields. It does not perform any
56 /// legality checks.
57 ///
58 /// Prerequisite: `newYieldedValues.size() == newYieldedValues.size()`.
59 scf::ForOp cloneWithNewYields(OpBuilder &b, scf::ForOp loop,
60  ValueRange newIterOperands,
61  ValueRange newYieldedValues,
62  bool replaceLoopResults = true);

64 /// Outline a region with a single block into a new FuncOp.
