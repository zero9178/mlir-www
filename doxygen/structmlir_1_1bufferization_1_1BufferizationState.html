Creates a memref allocation for the given shaped value.

Create an allocation after shapedValue.getDefiningOp (or at the top of the block in case of a bbArg).

dealloc indicates whether the buffer should be deallocated or not. When dealloc is false, this would create a memory leak, unless the buffer is deallocated through some other mechanism.

dealloc is optional. By default, this function will figure out by itself if it is safe to deallocate the buffer. In essence, when returning the buffer from a block, it is not safe to deallocate the buffer. This information is queried via AnalysisState::isTensorYielded.

Note: shapedValue is typically a tensor value. However, if it is a memref value, dealloc is no longer optional and must be specified.

Return the buffer (memref) for a given OpOperand (tensor).

Allocate a new buffer and copy over data from the existing buffer if out-of-place bufferization was decided.

Whether a buffer is in-place or out-of-place is queried from the analysis state. Some analyses may always conservatively opt for out-of-place bufferization. Inplacability decisions can be overridden with the optional overrideInPlace parameter.
